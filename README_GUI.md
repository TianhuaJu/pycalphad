# 合金液相线/固相线计算 GUI 工具

## 📋 功能特性

### 核心功能
- ✅ 加载热力学数据库（TDB格式）
- ✅ 多种模型选择（RKM, Muggianu, Toop, UEM1）
- ✅ 灵活的成分设定
- ✅ 可调节的温度范围
- ✅ 相选择
- ✅ 实时图形显示
- ✅ 数据表格查看
- ✅ 计算日志追踪
- ✅ 数据导出功能

### 支持的模型

| 模型 | 说明 | 适用场景 |
|------|------|---------|
| **RKM** | Redlich-Kister-Muggianu | 标准三元对称外推 |
| **Muggianu** | Muggianu几何外推 | 液相计算 |
| **Toop** | Toop非对称外推 | 液相计算（有渐近组分） |
| **UEM1** | 统一外推模型 | 基于无限稀释性质的外推 |


## 📖 使用指南

### 1. 加载数据库

1. 点击 **"加载数据库"** 按钮
2. 选择TDB格式的热力学数据库文件（如 `examples/alcrni.tdb`）
3. 状态会显示为 "已加载: xxx.tdb"（绿色）

### 2. 选择模型

从 **"模型选择"** 下拉菜单中选择：
- **RKM (Redlich-Kister-Muggianu)** - 默认，所有相使用传统模型
- **Muggianu** - 液相使用Muggianu，固相使用RKM
- **Toop** - 液相使用Toop，固相使用RKM
- **UEM1** - 液相使用UEM，固相使用RKM

### 3. 设定成分

#### 组分列表
输入参与计算的组分，逗号分隔（如 `AL,CR,NI`）

#### 扫描组分
选择作为横坐标的组分（如 `NI`）

#### 扫描范围
格式：`起始值,终止值,点数`
- 示例：`0.1,0.9,10` 表示从0.1到0.9，计算10个点

#### 其他组分比例
设定除扫描组分外的其他组分比例
- 格式：用冒号分隔的比例（如 `1:1` 表示等比例）
- 示例：对于AL-CR-NI系统，扫描NI时，设置 `1:1` 表示 X(AL) = X(CR)

### 4. 设定温度范围

- **最低温度**：搜索液相线的最低温度（K）
- **最高温度**：搜索液相线的最高温度（K）
- **温度步长**：计算步长（K），越小越精确但计算越慢

推荐值：
```
最低温度: 1400 K
最高温度: 2400 K
温度步长: 10 K
```

### 5. 选择相

输入要考虑的相，逗号分隔
- 示例：`LIQUID,FCC_A1,BCC_A2,B2,L12_FCC`
- 必须包含 `LIQUID` 相

### 6. 开始计算

1. 点击 **"开始计算"** 按钮
2. 计算过程会显示在 "计算日志" 标签页
3. 进度会实时更新
4. 完成后自动切换到 "图形结果" 标签页

### 7. 查看结果

#### 图形结果
- 液相线温度 vs 成分曲线
- 可使用matplotlib工具栏进行缩放、平移、保存等操作

#### 数据表格
- 切换到 "数据表格" 标签页查看详细数值
- 包含所有计算点的成分和液相线温度

#### 计算日志
- 切换到 "计算日志" 标签页查看计算过程
- 包含每个点的详细信息和可能的警告/错误

### 8. 导出数据

1. 点击 **"导出数据"** 按钮
2. 选择保存位置和文件名
3. 支持 .txt 和 .csv 格式

## 🔧 高级技巧

### 1. 调整温度范围提高精度

如果液相线在特定区间，可以缩小温度范围：
```
最低温度: 1600 K
最高温度: 1800 K
温度步长: 5 K
```

### 2. 处理计算失败

如果某些点计算失败（显示N/A）：
- 检查温度范围是否合适
- 尝试增加温度步长
- 检查成分是否在合理范围内
- 查看计算日志了解详细错误信息

### 3. 批量计算

对于需要多次计算的情况，建议使用命令行脚本：
```bash
python calculate_liquidus_alcrni.py
```

该脚本可以自动对比多个模型。

## ⚠️ 注意事项

1. **数据库兼容性**
   - 确保TDB文件格式正确
   - 组分名称必须与数据库中一致（区分大小写）

2. **计算时间**
   - 计算时间取决于点数和温度步长
   - 建议先用少量点测试（如5个点）
   - 确认无误后再增加点数

3. **模型选择**
   - UEM1需要确保已正确安装（修复了绝对值问题的版本）
   - 如果高级模型加载失败，会回退到RKM

4. **内存使用**
   - 大量计算点可能占用较多内存
   - 建议分批计算或减少点数

## 🐛 故障排除

### 问题1：无法加载数据库

**症状**：点击"加载数据库"后报错

**解决方案**：
1. 检查TDB文件路径是否正确
2. 确保文件不是损坏的
3. 尝试使用示例数据库 `examples/alcrni.tdb`

### 问题2：计算失败

**症状**：所有点都显示 N/A

**解决方案**：
1. 检查温度范围是否合理
2. 确认相列表包含LIQUID
3. 检查成分总和是否为1
4. 查看计算日志了解详细错误

### 问题3：图形不显示

**症状**：计算完成但图形为空

**解决方案**：
1. 切换到"图形结果"标签页
2. 点击"清除结果"后重新计算
3. 检查是否有有效数据点（非NaN）

### 问题4：程序无响应

**症状**：点击"开始计算"后程序卡住

**解决方案**：
1. 这是正常现象，计算在后台线程运行
2. 查看进度提示
3. 如果长时间无响应，重启程序并减少计算点数
