# 双数据库功能 - 用户指南

## 🎯 功能概述

新版本GUI支持**独立加载纯组元数据库和系统数据库**，实现了数据源的灵活管理。

### 核心优势

1. **高质量纯组元数据** - 使用专门的纯组元数据库（如unary50.tdb）获得更准确的纯元素热力学数据
2. **便于多系统扩展** - 同一个纯组元数据库可复用于多个系统（二元、三元、四元、五元等）
3. **数据管理清晰** - 纯组元数据和交互参数分离，方便维护和更新
4. **向后兼容** - 纯组元数据库是可选的，不加载时自动使用系统数据库的纯组元数据

## 📋 使用方法

### 方法1：仅使用系统数据库（传统方式）

**适用场景**: 系统数据库已包含可靠的纯组元数据

**操作步骤**:
1. 点击 **[加载系统TDB数据库 (必需)]**
2. 选择系统数据库文件（可多选）
3. 开始计算

**示例**:
```
[加载系统TDB数据库] → 选择: alcrni.tdb
```

**结果**:
- 纯组元数据库: 未加载 (将使用系统数据库的纯组元数据)
- 系统数据库: ✓ 已加载: alcrni.tdb
- 纯组元数据来源: alcrni.tdb

### 方法2：使用纯组元数据库 + 系统数据库（推荐）

**适用场景**:
- 需要高精度纯组元数据
- 计算多个不同系统
- 扩展到多元系统

**操作步骤**:
1. 点击 **[加载纯组元数据库 (可选)]**
2. 选择纯组元数据库（如unary50.tdb）
3. 点击 **[加载系统TDB数据库 (必需)]**
4. 选择系统数据库文件（可多选）
5. 系统自动合并两个数据库
6. 开始计算

**示例**:
```
[加载纯组元数据库] → 选择: unary50.tdb
[加载系统数据库]   → 选择: alcrni_interactions.tdb
```

**结果**:
- 纯组元数据库: ✓ 已加载: unary50.tdb
- 系统数据库: ✓ 已加载: alcrni_interactions.tdb
- 纯组元数据来源: unary50.tdb (高质量数据)
- 交互参数来源: alcrni_interactions.tdb

### 加载顺序说明

⚠️ **两个数据库的加载顺序不影响结果**

**情况1: 先加载纯组元数据库，后加载系统数据库**
```
1. [加载纯组元数据库] → unary50.tdb
2. [加载系统数据库]   → alcrni.tdb
   → 自动合并
```

**情况2: 先加载系统数据库，后加载纯组元数据库**
```
1. [加载系统数据库]   → alcrni.tdb
2. [加载纯组元数据库] → unary50.tdb
   → 检测到系统数据库已加载，自动重新合并
```

## 🔄 数据库合并机制

### 合并策略

当同时加载两个数据库时，系统执行以下合并操作：

```
最终数据库 = 系统数据库 + 纯组元数据库（覆盖）

具体操作：
├─ 1. 纯组元GHSER函数      → 用纯组元数据库覆盖
├─ 2. 参考态定义            → 用纯组元数据库覆盖
├─ 3. 纯组元G参数 (order=0) → 用纯组元数据库替换
├─ 4. 交互参数 (order>0)    → 保留系统数据库
├─ 5. 相定义                → 保留系统数据库
└─ 6. 元素和物种            → 两者合并
```

### 合并日志示例

```
✓ 纯组元数据库加载成功: unary50.tdb
  - 包含 50 个元素
  - 包含 150 个函数定义 (如GHSER等)

✓ 系统数据库加载成功: alcrni.tdb
  - 包含 3 个元素
  - 包含 5 个相

开始合并数据库...
  ✓ 覆盖了 3 个纯组元GHSER函数
  ✓ 替换了 15 个纯组元G参数
✓ 数据库合并完成 (纯组元数据来自纯组元数据库)

可用组分 (3): ['AL', 'CR', 'NI']
可用相 (5): ['BCC_A2', 'FCC_A1', 'LIQUID', 'SIGMA', ...]
```

## 💾 配置持久化

### 目录记忆功能

系统分别记住两个数据库的最后打开目录：

**配置文件**: `~/.pycalphad_gui_config.json`

```json
{
  "last_unary_directory": "/path/to/unary_databases",
  "last_system_directory": "/path/to/system_databases"
}
```

**优点**:
- 纯组元数据库和系统数据库可以存储在不同目录
- 每次打开对话框自动定位到相应目录
- 提高工作效率

## 🎓 应用场景

### 场景1: 单一系统研究

**需求**: 研究 Al-Cr-Ni 三元系统

**方案**: 仅加载系统数据库
```
[加载系统数据库] → alcrni.tdb
```

### 场景2: 多系统研究（推荐）

**需求**: 研究多个系统 (Al-Cr-Ni, Al-Fe-Co, Ni-Fe-Mo)

**方案**: 使用统一的纯组元数据库
```
第一次:
[加载纯组元数据库] → unary50.tdb (仅需加载一次)
[加载系统数据库]   → alcrni_interactions.tdb

第二次:
[加载系统数据库]   → alfeco_interactions.tdb
(纯组元数据库已加载，无需重新加载)

第三次:
[加载系统数据库]   → nifemo_interactions.tdb
(纯组元数据库已加载，无需重新加载)
```

**优势**:
- 纯组元数据一致性
- 避免重复加载
- 便于比较不同系统

### 场景3: 扩展到高阶系统

**需求**: 从二元 → 三元 → 四元 → 五元系统研究

**方案**: 建立数据库体系
```
纯组元数据库 (基础):
  unary50.tdb (50个元素的高质量纯组元数据)

系统数据库 (按需加载):
  ├─ binary/
  │  ├─ al-cr.tdb
  │  ├─ al-ni.tdb
  │  └─ cr-ni.tdb
  │
  ├─ ternary/
  │  ├─ alcrni.tdb
  │  └─ alfecr.tdb
  │
  ├─ quaternary/
  │  └─ alcrfeni.tdb
  │
  └─ quinary/
     └─ alcrfeconi.tdb (高熵合金)

使用时:
[加载纯组元数据库] → unary50.tdb
[加载系统数据库]   → 选择相应的系统数据库
```

## ⚙️ 技术细节

### 纯组元数据优先级

```python
# 纯组元数据的优先级顺序：
1. 纯组元数据库 (如果加载)     ← 最高优先级
2. 系统数据库                   ← 备用
```

### 参数类型说明

| 参数类型 | 来源 | 说明 |
|---------|------|------|
| **GHSERFE, GHSERNI** (FUNCTION) | 纯组元数据库 | 纯元素标准态Gibbs能量函数 |
| **G(FCC,AL:VA;0)** (纯组元G) | 纯组元数据库 | 纯组元在特定相中的Gibbs能量 |
| **L(LIQUID,AL,CR;0)** (交互参数) | 系统数据库 | 二元、三元交互参数 |
| **相定义 (FCC, BCC, LIQUID)** | 系统数据库 | 晶体结构和亚晶格模型 |

### 模型与参考态

⚠️ **重要**: 参考态是模型无关的

- **参考态** (纯组元化学势): 由数据库中的纯组元参数决定，与模型无关
- **混合态**: 使用自定义模型（UEM、Muggianu、Toop等）
- **活度计算**: a = exp((μ_混合 - μ_参考) / RT)

详见文档: `PYCALPHAD_DATABASE_MECHANISM.md`

## 🔍 故障排除

### 问题1: "加载纯组元数据库失败"

**可能原因**:
- TDB文件编码问题
- 文件格式错误

**解决方案**:
1. 检查文件是否为有效的TDB格式
2. 尝试使用文本编辑器检查文件编码（应为latin-1或utf-8）
3. 查看日志输出获取详细错误信息

### 问题2: "合并数据库失败"

**可能原因**:
- 纯组元数据库和系统数据库元素不匹配
- 内存不足

**解决方案**:
1. 检查两个数据库是否包含相同的元素
2. 查看日志输出获取详细错误信息
3. 尝试分别加载两个数据库，确认各自可以单独使用

### 问题3: "计算结果与预期不符"

**检查清单**:
1. ✅ 确认两个数据库都已成功加载（标签显示绿色）
2. ✅ 查看日志，确认合并操作成功
3. ✅ 检查覆盖的GHSER函数数量是否正确
4. ✅ 对比单独使用系统数据库和使用合并数据库的结果差异
