# 快速开始 - 合金液相线GUI

## 🚀 5分钟快速上手

### 1. 安装依赖（首次使用）
```bash
pip install pycalphad numpy matplotlib
```

### 2. 启动程序
```bash
# Linux/Mac
./run_gui.sh

# Windows
run_gui.bat

# 或直接运行
python phase_diagram_gui.py
```

### 3. 第一次计算（3步骤）

#### 步骤1：加载数据库
1. 点击左上角 **"加载数据库"** 按钮
2. 选择 `examples/alcrni.tdb`
3. 看到绿色 "已加载" 提示 ✅

#### 步骤2：检查默认设置
默认设置已经配置好Al-Cr-Ni系统：
```
✓ 组分列表: AL,CR,NI
✓ 扫描组分: NI
✓ 扫描范围: 0.1,0.9,10
✓ 其他组分比例: 1:1 (AL:CR = 1:1)
✓ 温度范围: 1400-2400 K
✓ 相列表: LIQUID,FCC_A1,BCC_A2,B2,L12_FCC
✓ 模型: RKM
```

#### 步骤3：开始计算
1. 点击 **"开始计算"** 按钮
2. 等待1-2分钟（计算10个点）
3. 查看结果！

### 4. 查看结果（3个标签页）

#### 📊 图形结果
- 自动显示液相线曲线
- 可以缩放、平移、保存图片
- 横轴：X(NI)，纵轴：T_liquidus (K)

#### 📋 数据表格
- 切换到 "数据表格" 查看数值
- 包含每个点的成分和液相线温度
- 可以复制到Excel

#### 📝 计算日志
- 查看计算过程
- 检查是否有警告或错误
- 了解每个点的详细信息

## 💡 常用操作

### 比较不同模型
1. 完成第一次计算后，记录结果
2. 在"模型选择"中选择 **UEM1**
3. 再次点击"开始计算"
4. 对比两次结果

### 更改成分比例
**示例：X(AL):X(CR) = 2:1**
```
组分列表: AL,CR,NI
扫描组分: NI
其他组分比例: 2:1  ← 修改这里
```

### 调整计算精度
**快速计算（5个点）：**
```
扫描范围: 0.1,0.9,5
```

**高精度计算（20个点）：**
```
扫描范围: 0.1,0.9,20
温度步长: 5  ← 更精确
```

### 导出数据
1. 计算完成后，点击 **"导出数据"**
2. 选择保存位置（支持.txt或.csv）
3. 数据可在Excel中打开

## 🎯 典型应用场景

### 场景1：研究模型差异
```
目标：比较RKM和UEM1模型对Al-Cr-Ni液相线的预测
操作：
1. 使用RKM计算并记录结果
2. 使用UEM1计算并对比
3. 导出两组数据进行分析
```

### 场景2：优化合金成分
```
目标：找到特定液相线温度的成分
操作：
1. 设置较密的扫描点（如20个点）
2. 查看图形找到目标温度
3. 从数据表格中读取对应成分
```

### 场景3：三元相图截面
```
目标：绘制不同成分比例的液相线
操作：
1. 第一次：比例1:1，导出数据data1.txt
2. 第二次：比例2:1，导出数据data2.txt
3. 第三次：比例1:2，导出数据data3.txt
4. 在外部软件中综合绘图
```

## ⚠️ 常见问题

### Q: 计算时程序卡住了
**A:** 这是正常的！计算在后台进行，查看进度提示。如果点数多（>20），可能需要几分钟。

### Q: 所有点都显示 N/A
**A:** 检查：
1. 温度范围是否太窄（建议1400-2400K）
2. 相列表是否包含LIQUID
3. 查看"计算日志"了解详细错误

### Q: 某些点计算失败
**A:** 这可能是正常的（某些成分可能没有稳定的液相），继续查看其他点的结果。

### Q: 图形不显示
**A:**
1. 切换到"图形结果"标签页
2. 确认有有效数据点（不是全部N/A）
3. 尝试"清除结果"后重新计算

### Q: 想要更高精度
**A:**
```
温度步长: 5 (从10改为5)
扫描点数: 20 (从10改为20)
```
注意：计算时间会成倍增加
