## 问题 2: 输入面板优化 ✅ 已完成

**用户问题**: 优化一下输入面板，使其逻辑性更强，更易操作。

  - 提供更清晰的错误提示

**修改位置**:

- `_parse_inputs()`: 从三个字段读取并验证数据 (lines 578-593)

## 问题 3: 活度计算 Bug ✅ 已修复

**用户问题**: 活度计算部分，结果无法显示，存在 bug。

**根本原因**:
1. 参考态化学势计算失败：纯组分在液相中可能不稳定
2. 化学势提取不稳定：使用 `.squeeze().item()` 可能失败
3. 缺少数据验证：NaN 值未被检测
4. 错误信息不明确：绘图失败时只显示 "No valid data to plot"

**解决方案**:

### 1. 改进参考态化学势计算 (`_calculate_reference_potentials`, lines 1081-1158)

**新增功能**:
- **双重计算策略**:
  - **方法 1**: 优先尝试液相计算
  - **方法 2**: 若液相失败，自动回退到稳定相计算

- **安全的数据提取**:
  ```python
  mu_data = ref_eq.MU.sel(component=comp)
  if mu_data.size == 0:
      raise ValueError(f"{comp} 在 {liquid_phase_name} 中无 MU 数据")
  mu_ref = float(mu_data.values.flatten()[0])
  ```

- **完整的数据验证**:
  ```python
  if np.isnan(mu_ref) or np.isinf(mu_ref):
      raise ValueError(f"{comp} MU 值无效 (NaN/Inf)")
  ```

- **详细的日志输出**:
  ```
  ✓ MU_ref(AL) = -7842.15 J/mol  [方法: liquid(LIQUID)]
  ✗ CR 所有方法均失败，设置为 NaN
  ```

### 2. 改进活度计算 (lines 1033-1063)

**新增功能**:
- **安全的化学势提取**:
  ```python
  mu_data = eq.MU.sel(component=comp)
  if mu_data.size == 0:
      raise ValueError(f"{comp} 无 MU 数据")
  mu_mix = float(mu_data.values.flatten()[0])
  ```

- **多层数据验证**:
  - 验证化学势值（NaN/Inf）
  - 验证活度值（NaN/Inf/负数）

- **智能错误日志**: 只在扫描范围边界记录错误（避免日志过多）
  ```python
  if x_scan < 0.15 or x_scan > 0.85:
      self.log(f"    {comp} 活度计算失败 (X={x_scan:.3f}): {e}")
  ```

### 3. 改进错误提示 (`_plot_activity`, lines 1274-1286)

**原来的提示**:
```
No valid data to plot
```

**新的详细提示**:
```
❌ 无有效活度数据

可能原因:
• 参考态化学势计算失败
• 混合相化学势提取失败
• 所有数据点计算失败

请查看 [日志] 标签页获取详细信息
```

带有：
- 红色边框和背景高亮
- 明确的错误图标
- 清晰的可能原因列表
- 指引用户查看详细日志
