# 合金热力学计算与模型对比工具 - 使用指南

## 📋 概述

这是一个**全新设计的简化版GUI工具**，专注于：
- ✅ **多模型对比**：RKM, Muggianu, Toop, UEM1
- ✅ **自动化配置**：自动识别数据库中的组分和相
- ✅ **热力学性质**：计算自由能、活度等
- ✅ **相图生成**：自动计算并保存相图
- ✅ **结果对比**：直观对比不同模型的预测

## 🚀 快速启动

```bash
python alloy_calculator_gui.py
```

## 🎯 核心功能

### 1. 液相线多模型对比
- 同时计算多个模型的液相线
- 在同一图上对比不同模型的预测
- 自动识别液相，无需手动指定相名称

### 2. 热力学性质计算
- **Gibbs自由能** vs 成分
- **活度** vs 成分
- 支持多模型对比

### 3. 相图生成
- 自动计算温度-成分相图
- 以PNG格式保存到当前目录
- 文件名包含模型名称和时间戳

## 📖 详细使用步骤

### 步骤1：加载数据库

1. 点击 **"加载TDB数据库"** 按钮
2. 选择您的TDB文件（如 `Al-Cu-Y.tdb`）
3. 界面会自动显示：
   - ✅ 可用组分（如：AL, CU, Y）
   - ✅ 可用相（如：LIQUID, FCC_A1, BCC_A2等）
   - ✅ 成功加载的模型

**重要**：程序会自动识别所有相，无需手动输入！

### 步骤2：设置计算参数

#### 2.1 成分设置
```
扫描组分: Y              ← 要扫描的组分
扫描范围: 0.1,0.9,10     ← 起始值,终止值,点数
其他组分比例: 1:1        ← AL:CU = 1:1
```

**示例成分计算**：
- X(Y) = 0.1, X(AL) = 0.45, X(CU) = 0.45
- X(Y) = 0.5, X(AL) = 0.25, X(CU) = 0.25
- X(Y) = 0.9, X(AL) = 0.05, X(CU) = 0.05

#### 2.2 温度设置
```
最低温度: 300 K
最高温度: 2000 K
温度步长: 10 K
```

**注意**：
- 液相线计算会扫描整个温度范围
- 性质计算使用"最高温度"作为计算温度
- 步长越小，计算越精确，但时间越长

#### 2.3 模型选择

勾选您想要对比的模型（可多选）：
- ☑ **RKM** - Redlich-Kister-Muggianu（标准模型）
- ☑ **Muggianu** - 几何外推模型
- ☑ **Toop** - 非对称外推模型
- ☑ **UEM1** - 统一外推模型

**自动应用规则**：
- 液相（LIQUID）→ 使用您选择的模型
- 固相（FCC_A1, BCC_A2等）→ 自动使用RKM

### 步骤3：执行计算

#### 3.1 计算液相线对比

1. 点击 **"计算液相线对比"** 按钮
2. 程序自动对每个选中的模型计算液相线
3. 结果显示在 **"液相线对比"** 标签页
4. 查看 **"日志"** 标签页了解详细计算过程

**结果图特点**：
- 不同模型用不同颜色和标记
- RKM: 蓝色圆圈
- Muggianu: 橙色方块
- Toop: 绿色三角
- UEM1: 红色倒三角

#### 3.2 计算热力学性质

1. 点击 **"计算热力学性质"** 按钮
2. 程序计算Gibbs自由能和活度
3. 结果显示在 **"热力学性质"** 标签页
   - 子标签 **"自由能"**: Gibbs能 vs 成分
   - 子标签 **"活度"**: 各组分活度 vs 成分

**计算温度**：使用"最高温度"设置值

#### 3.3 生成相图

1. 点击 **"生成相图"** 按钮
2. 程序计算温度-成分相图（需要几分钟）
3. 结果显示在 **"相图"** 标签页
4. **自动保存**为PNG图片到当前目录

**文件命名示例**：
```
phase_diagram_RKM_20251025_143522.png
phase_diagram_UEM1_20251025_143658.png
```

### 步骤4：查看和导出结果

#### 4.1 查看数据表格

切换到 **"数据"** 标签页，查看详细数值：

```
X(Y)      T_liq_RKM (K)    T_liq_Muggianu (K)    T_liq_UEM1 (K)
========================================================================
0.1000    1520.00          1515.30               1518.50
0.2000    1480.00          1476.20               1479.10
...
```

#### 4.2 导出数据

1. 在 **"数据"** 标签页点击 **"导出数据"** 按钮
2. 选择保存位置和格式（.txt 或 .csv）
3. 可在Excel中打开分析

#### 4.3 保存图片

使用matplotlib工具栏：
- 🖼️ **保存图标**：保存当前图形
- 🔍 **缩放工具**：放大查看细节
- 🏠 **主页按钮**：重置视图

1. 只勾选 ☑ RKM 和 ☑ UEM1
2. 计算液相线对比
3. 观察曲线差异
4. 计算热力学性质，对比自由能

### 技巧4：检查计算是否合理

查看 **"日志"** 标签页：
- ✅ 如果看到具体的温度值 → 计算成功
- ⚠️ 如果看到"未找到" → 该成分点可能超出液相区

**正常日志示例**：
```
[14:35:22] 开始计算: RKM
[14:35:23] X(Y)=0.100, X(AL)=0.450, X(CU)=0.450 -> T_liq = 1520.0 K
[14:35:24] X(Y)=0.200, X(AL)=0.400, X(CU)=0.400 -> T_liq = 1480.0 K
```

**异常日志示例**：
```
[14:35:22] X(Y)=0.100, X(AL)=0.450, X(CU)=0.450 -> T_liq = 300.0 K  ← 太低！
```
这说明温度范围设置不当或数据库有问题。

## 🔧 问题排查

### 问题1：所有液相线温度都相同（如都是300 K）

**原因**：温度范围设置不当

**解决**：
1. 检查最低温度和最高温度
2. 对于金属合金，建议：
   ```
   最低温度: 300 K
   最高温度: 2000 K
   ```
3. 对于高熔点合金，可能需要：
   ```
   最高温度: 3000 K
   ```

### 问题2：部分模型不可用

**现象**：日志显示"警告: Muggianu 模型未加载，跳过"

**原因**：高级模型文件缺失

**解决**：
1. 确认 `pycalphad/advanced_uem_model.py` 存在
2. 确认 `pycalphad/uem1_Model.py` 存在
3. 如果文件存在但仍无法加载，检查导入错误

### 问题3：相图计算时间过长

**原因**：计算点数太多

**解决**：
1. 减少成分点数：
   ```
   扫描范围: 0.1,0.9,5   ← 从10减到5
   ```
2. 减少温度精度：
   ```
   代码中 temp_num = 50 → 改为 temp_num = 20
   ```

### 问题4：活度显示为NaN

**原因**：数据库可能不支持活度计算

**解决**：
- 这是正常的，某些数据库格式不包含活度数据
- 专注于Gibbs自由能分析

### 问题5：程序卡住不动

**现象**：点击计算后界面无响应

**原因**：计算在后台进行

**解决**：
1. 查看"进度"提示（左下角）
2. 查看"日志"标签页了解进度
3. 液相线计算：约1-3分钟（10个点）
4. 相图计算：约5-10分钟

## ⚙️ 技术细节

### 液相线查找算法（已修复）

**问题**：原版本所有液相线都返回最低温度

**原因**：循环逻辑错误，从高温向低温扫描但没有正确break

**修复后的逻辑**：
```python
# 从低温向高温扫描，找到最后一个100%液相的点
for t_idx in range(len(T_vals)):
    liquid_np = 0.0

    # 查找LIQUID相的分数
    for v_idx in range(phase_array.shape[-1]):
        if 'LIQUID' in phase_name:
            liquid_np = np_array[...][v_idx]
            break

    # 如果液相分数>=99.5%，更新液相线温度
    if liquid_np >= 0.995:
        liquidus_T = T_vals[t_idx]
    # 如果<99.5%且之前找到过100%液相，说明已过液相线，退出
    elif liquidus_T is not None:
        break
```
